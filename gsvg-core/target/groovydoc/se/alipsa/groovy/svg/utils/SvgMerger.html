[Groovy] Class SvgMerger

    • se.alipsa.groovy.svg.utils.SvgMerger 
    • @groovy.transform.CompileStatic
class SvgMerger
extends java.lang.Object

      Utility for merging multiple SVG documents using a pure object-oriented approach.

      Pure OO Implementation

      This merger uses SvgElementFactory for efficient element copying without XML serialization. The factory populates both DOM4J structures and SvgElement children lists in a single pass, eliminating the need for XML parsing after merge operations.

      Performance

      The pure OO approach provides:
      • Zero XML serialization overhead during merge operations
      • Direct object manipulation for better performance
      • Single-pass construction maintains both DOM and object model consistency
