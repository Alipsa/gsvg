[Groovy] Class SvgElement<T extends SvgElement<T>>

    • se.alipsa.groovy.svg.SvgElement
    • All Implemented Interfaces and Traits:
      ElementContainer, java.lang.Cloneable
      @groovy.transform.CompileStatic
abstract class SvgElement<T extends SvgElement<T>>
extends java.lang.Object
implements ElementContainer, java.lang.Cloneable

      Base class for all SVG elements backed by DOM4J elements.

      Constructor Pattern

      Each concrete SvgElement subclass provides two types of constructors:
      1. Creating constructors - Create new DOM elements and add them to the parent: 
        ClassName(SvgElement parent)
        These constructors call super(parent, NAME) to create a new DOM element.
      2. Adopting constructors - Adopt existing DOM elements for cloning/copying: 
        ClassName(SvgElement parent, Element element)
        These constructors call super(parent, element) to wrap an existing DOM element. Used by SvgElementFactory for pure object-oriented element copying without XML serialization.

      Cloning

      All SvgElement instances can be cloned using the clone(AbstractElementContainer) method, which creates a deep copy including all child elements and attributes. The cloning uses the pure object-oriented approach without XML serialization for optimal performance.

      Example:

       Circle original = svg.addCircle().cx(50).cy(50).r(25)
 Circle copy = original.clone(targetSvg)
      The adopting constructor pattern enables efficient deep copying of SVG element trees while maintaining both the DOM4J structure and the SvgElement object hierarchy.
      See Also:
      SvgElementFactory.deepCopy

        • T clone(AbstractElementContainer newParent)

          Creates a deep clone of this element with all its children and attributes. This is the recommended way to clone SVG elements as it uses the pure object-oriented approach without XML serialization for optimal performance.

          The clone will be attached to the specified parent container and will include:

          • All attributes from the original element
          • All child elements (recursively cloned)
          • Proper parent-child relationships in both DOM and object model

          Example:

           // Create an original element with children
 G originalGroup = svg.addG().id("original")
 originalGroup.addCircle().cx(10).cy(10).r(5)
 originalGroup.addRect().x(20).y(20).width(10).height(10)

 // Clone it to another SVG
 G clonedGroup = originalGroup.clone(anotherSvg)
 // clonedGroup now has the same structure and children as originalGroup
          Parameters:
          newParent - the parent container for the cloned element
          Returns:
          a deep clone of this element with type safety preserved
          See Also:
          SvgElementFactory.deepCopy

          Parameters:

        • @java.lang.Override
          void setProperty(java.lang.String propertyName, java.lang.Object newValue)

          Overrides Groovy's property assignment to provide automatic fallback to addAttribute().

          This allows intuitive property-style assignment to SVG attributes. When assigning to a property that doesn't have an explicit setter (e.g., element.values = '0.5'), this method will automatically set it as an attribute using addAttribute('values', '0.5').

          This maintains backward compatibility with existing explicit setters while providing a more natural API for setting arbitrary SVG attributes.

          Note: Use with care - typos in property names will silently create incorrect attributes rather than failing at compile time. For critical code, consider using the explicit fluent API methods (e.g., .gradientUnits('value')) which provide better IDE autocomplete support.

          Parameters:
          propertyName - the name of the property to set
          newValue - the value to assign
